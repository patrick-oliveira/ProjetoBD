A modelagem das entregas foi feita reunindo os dados de diferentes tipos de entregas (medicamentos, respiradores e outros ítens). Cada tabela, sendo relativa a um tipo de entrega diferente, diferia das outras em alguns atributos, e o modelo foi feito de modo a tentar sintetizar todas estas tabelas em uma estrutura só. Para isso, separou-se os dados dos ítens do registro das entregas, assim seria possível criar uma base de dados de todos os tipos diferentes de ítens que foram entregues e cada registro de entrega teria as informações de quantidade, valor, etc., apenas fazendo referência ao ítem entregue.

Originalmente, algumas entregas estavam destinadas a apenas um estado em específico, enquanto outras possuíam informações mais detalhadas, especificando também o município de entrega ou até mesmo o destinatário. Os campos para informações mais detalhadas foram incluídas no modelo, admitindo que em alguns casos estes campos permaneceriam nulos. As informações de destino foram colocadas na relação "Entrega Realizada" que faz o cruzamento entre os códigos de cada registro de entrega e o seu destino. 


Entidade Federativa

ID: INT (chave estrangeira)

município:
ID: INT (chave estrangeira)
estado: INT (chave estrangeira)

Os municípios, estados e países foram modelados como especializações de uma entidade "Entidade Federativa", que possui os atributos comuns destes três tipos. A reunião destes três tipos em uma única entidade é necessária para simplificar o modelo, que passa a levar em conta o fato de que algumas entidades podem referenciar, por exemplo, um estado ou um município. Este é o caso das informações de entrega - no modelo, entende-se que cada entrega é feita para uma entidade federativa, e o tipo do destino dependerá da informação cadastrada. Um outro exemplo se dá nas informações sobre casos e óbitos, onde alguns dados dizem respeito aos países e outros aos estados ou municípios.
